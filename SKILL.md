# 富途 CRS 境外所得税计算

## 概述

此技能用于解析富途( Futu )下载的年度账单 Excel 文件，按照中国个人所得税法规定，
计算境外股票/基金/期权转让所得、股息红利、利息收入，最终确定需要在个税 APP 中
申报的应补税额。

核心方法采用**移动加权平均法**计算持仓成本和卖出盈亏(P/Q/R列)，
与财税〔2016〕36号文的要求一致。

## 使用前准备

用户需要从富途 App 下载年度账单：
1. 富途 App → 我的账户 → 全部 → 我的税表 → 选择对应年度
2. 下载"自动交换财务账户资料(CRS)"对应的 Excel 文件
3. 如果持有跨年度持仓，建议同时下载上一年度的账单

## 工作流程

### 第一步：确认输入文件

询问用户：
- 当年度的富途年度账单 Excel 文件路径
- 是否有上年度账单（用于期初持仓成本结转）
- 纳税年度（如不提供，从文件名自动提取）
- 是否需要手动指定汇率（如不提供，自动从网络获取）

### 第二步：运行计算脚本

核心计算由 `scripts/calculate_tax.py` 完成。运行命令：

```bash
python3 <skill-path>/scripts/calculate_tax.py <年度账单.xlsx> [上年度账单.xlsx] [-o 输出文件.xlsx]
```

可选参数：
- `--year 2025`：手动指定纳税年度
- `--usd-rate 7.1`：手动指定 USD/CNY 汇率
- `--hkd-rate 0.91`：手动指定 HKD/CNY 汇率

### 第三步：解读结果

脚本会输出三个核心数字：
1. **资本利得税**：卖出盈利 × 20%（同市场同年度盈亏可互抵）
2. **股息红利税**：股息总额 × 20% − 境外已预扣税额
3. **利息收入税**：利息/票息收入 × 20%

最终合计 = 1 + 2 + 3，即需要在个税 APP 中申报的应补税额。

## 计算逻辑详解

### 交易流水列（原始列）

富途年度账单 `证券-交易流水` sheet 包含 14 列：

| 列号 | 列名 | 说明 |
|------|------|------|
| A | 成交时间 | 交易时间戳 |
| B | 账户名称 | 如"保證金綜合帳戶(3385) - 證券" |
| C | 账户号码 | 如 1001265391083385 |
| D | 品类 | 证券/基金/期权/债券/结构化票据 |
| E | 代码名称 | 股票代码或基金代码 |
| F | 交易所/市场 | US=美股, SEHK=港股, FD=基金, -=不适用 |
| G | 方向 | 买入开仓/卖出平仓/申购/赎回/买入平仓/卖出开仓 |
| H | 交收日期 | T+2 结算日期 |
| I | 币种 | USD/HKD |
| J | 数量/面值 | 正=买入，负=卖出 |
| K | 价格 | 成交单价 |
| L | 成交金额 | 数量×价格 |
| M | 总费用 | 佣金+印花税+交易费 |
| N | 变动金额 | 成交金额+费用后的净变动 |

### P/Q/R 列（新增计算列）

#### P 列 — 剩余股数

累计持仓股数的滚动求和：
- 买入：P_after = P_before + qty
- 卖出：P_after = P_before − qty
- 归零后重新开始（本轮交易完结）

#### Q 列 — 加权平均成本（每股）

采用移动加权平均法，**买入手续费计入成本**（依据《个人所得税法》财产原值定义）：

```
买入时：new_cost = old_total_cost + (qty × price + buy_fee)
        new_Q = new_cost / new_P
卖出时：Q 不变（剩下的持仓成本保持不变）
清仓时：Q = 0
```

期初持仓：从上年度持仓总览或上年度计算结果的 Q 值结转。

#### R 列 — 卖出已实现盈亏

仅对卖出操作计算：

```
R = (卖出价格 − 加权成本Q) × 卖出股数 − 卖出手续费
```

- 正值 = 盈利，负值 = 亏损
- 买入操作的 R = 0
- **R列总和 × 20% = 非股息部分的资本利得税**

### 交易方向识别

| 方向 | 操作类型 | 影响 |
|------|---------|------|
| 买入开仓 | 买股票/基金(做多) | 买入，更新P/Q |
| 卖出平仓 | 卖股票/基金(平多) | 卖出，计算R，更新P |
| 申购 | 基金申购 | 买入，更新P/Q |
| 赎回 | 基金赎回 | 卖出，计算R，更新P |
| 买入平仓 | 期权平空仓 | 买入，更新P/Q |
| 卖出开仓 | 期权开空仓 | 卖出，计算R，更新P |

### 非交易获得的资产处理

通过 `证券-资产进出` sheet 识别以下类型：

1. **港股IPO公开发售**（IPO中签）
   - 方向：In
   - 成本：从 `证券-资金进出` 中查找对应 IPO 扣款记录
   - 如找不到，提示用户手动输入 IPO 发行价

2. **长期激励/股票收益计划**（RSU归属）
   - 方向：In
   - 成本：RSU归属日（vesting date）的股票市价
   - ⚠️ RSU归属时已按工资薪金缴税（雇主扣缴），卖出时以 vesting 市价为成本计算资本利得
   - Excel 中通常无 vesting 价格，需要用户根据归属日查找市价

3. **结构化票据**（如FCN）
   - 成本按票面金额
   - 票息收入归入利息收入类

### 股息与利息处理

从 `证券-资金进出` sheet 中识别：

- **股息收入**（公司行动 + DIVIDENDS）：计入股息总额
- **股息预扣税**（WITHHOLDING TAX）：计入境外已缴税额，可申请抵免
- **利息扣除**（月度利息扣除 Out）：这是融资利息支出，不影响应税计算
- **Note 票息**（Note Coupon/Knock-Out）：计入利息收入

### 汇率折算

- 汇率基准：纳税年度最后一日(12月31日)的人民币汇率中间价
- 数据来源：优先从 exchangerate-api 自动获取，失败则提示用户手动输入
- 所有外币金额统一折算为 RMB 后计税

### 税务计算

#### 资本利得税
```
应纳税所得额 = ∑(卖出盈亏)  （同市场同币种盈亏互抵）
应纳税额 = max(0, 应纳税所得额 × 汇率 × 20%)
```
- 同一市场（US/HK）同币种的盈亏可在年内互抵
- 亏损不能跨年度结转
- 浮盈不征税（仅已实现盈亏）

#### 股息红利税
```
应缴税额 = 股息总额(含预扣税前) × 汇率 × 20%
境外预扣税 = 实际被扣缴的税额
应补税额 = 应缴税额 − 境外预扣税（不可为负）
```

#### 利息收入税
```
应纳税额 = 利息/票息收入总额 × 汇率 × 20%
```

## 输出文件结构

生成的增强 Excel 包含 4 个 sheet：

1. **交易流水-含PQR**：原始交易记录 + P/Q/R 列，R列盈利绿色/亏损红色
2. **持仓汇总**：每个证券的期末持仓、成本、累计盈亏
3. **税务汇总**：资本利得、股息、利息的详细分项和合计应补税额
4. **股息明细**：股息收入及预扣税的逐笔记录

## 特殊情况处理

### 跨年度持仓
- 如果同一只股票上年度买入、本年度卖出，需要上年度文件来结转成本
- 如果用户未提供上年度文件，期初持仓成本默认为0（会导致税负偏高）
- ⚠️ **务必提醒用户**：没有期初成本数据会多算税

### 多账户
- 默认每个账户分开计算和展示
- 同一个人的不同账户最终应合并申报
- 账户间资金互转不影响税务计算（非应税事件）

### 期权
- 期权权利金收入按财产转让所得计税
- 卖出开仓收到权利金 → 买方行权或到期 → 计税
- 买入平仓支付权利金 → 亏损可抵扣

### 期货
- 如果用户有期货交易(在`期货-交易明细` sheet)，需要单独处理
- 默认不包含期货数据，需用户明确要求才处理

### 货币基金/余额宝类产品
- 基金申购/赎回的盈亏计入资本利得
- 货币基金的收益可能更适合作为利息收入，需具体情况判断

## 重要提醒（每次计算后提醒用户）

1. ⚠️ **申报时间**：次年3月1日至6月30日，逾期按日加收万分之五滞纳金（年化约18.25%）
2. 💡 **浮盈不征税**：仅对实际卖出实现的盈利计税，未卖出持仓无需缴税
3. 💡 **亏损不能跨年**：跨年度亏损不能结转抵扣
4. 💡 **股息预扣税可抵免**：美股股息通常预扣10%(W8BEN协议)，申报时填写境外已缴税额
5. 💡 **个税APP路径**：综合所得年度汇算 → 取得了境外所得 → 填写财产转让所得和利息股息红利所得
6. 💡 **RSU注意**：RSU归属时已按工资薪金缴税，卖出时以vesting市价为成本
7. 💡 **保留凭证**：建议保留券商年度对账单和完税证明至少5年

## 已知限制

- 汇率使用年度最后一日中间价，可能与申报时的实际汇率有微小差异
- 部分非交易获得的资产成本需要用户手动补充
- 结构化产品(FCN等)的税务处理可能因产品结构不同而异
- 期权交易的盈亏计算较为简化，复杂期权策略建议咨询专业税务师
